## 📊 对比分析：`search_arxiv_id_by_title` vs `search_arxiv_id_by_title_tfidf`

### 🧩 1. 概述

| 函数名称                       | 核心策略                              | 数据来源             | 匹配方式                                 |
|------------------------------|---------------------------------------|----------------------|------------------------------------------|
| `search_arxiv_id_by_title`   | Web 页面解析 + 精确字符串匹配         | arxiv.org 网页搜索    | 原始 HTML 提取标题后做清洗匹配           |
| `search_arxiv_id_by_title_tfidf` | API + 关键词查询 + TF-IDF + Cosine 相似度 | ArXiv API（XML 格式） | 使用语义特征向量进行模糊匹配             |

---

### 📌 2. 核心差异

| 比较项         | `search_arxiv_id_by_title`              | `search_arxiv_id_by_title_tfidf`           |
|----------------|-----------------------------------------|--------------------------------------------|
| **访问方式**    | 网页 HTML 请求                         | 官方 ArXiv API                             |
| **解析方式**    | 使用 `BeautifulSoup` 解析网页结构       | 使用 `ElementTree` 解析 XML 数据           |
| **查询精度**    | 完全依赖 `title == title` 的字符相等    | 使用关键词 + 向量相似度度量语义相似性      |
| **查询构造**    | 直接将 title 填入 URL 参数中            | 提取关键词，构造 `ti:关键词` AND 查询      |
| **容错性**      | 非常低，标题稍有不同就匹配失败          | 高容错，相似标题也能匹配成功                |
| **性能效率**    | HTML 结构易变，效率较低                | API 响应更快且结构稳定                      |

---
